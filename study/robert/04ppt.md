# 왜? ROS 2로 가야 하는가

## 📊 소스 분석 요약

- 핵심 메시지: ROS 2는 단순 업그레이드가 아닌 구조적 재설계의 결과
- 대상 청중: ROS 2 입문자 및 전환을 고민하는 개발자
- 포맷: Presenter Slides
- 총 슬라이드 수: 10장
- 예상 발표 시간: 15분

### 슬라이드 1: 왜? ROS 2로 가야 하는가

**슬라이드 내용**
- **왜? ROS 2로 가야 하는가**

### 슬라이드 2: 목차

**슬라이드 내용**
- 변화하는 로봇 개발 환경
- ROS 1의 구조적 한계
- ROS 2가 해결한 핵심 문제
- 지금 ROS 2를 선택해야 하는 이유
- 학습자·기업 관점의 장점

**시각적 제안**
- 레이아웃: 번호 리스트 + 아이콘
- 시각 요소: 흐름 화살표로 논리 연결

---
### 슬라이드 3: 변화하는 로봇 개발 환경

**슬라이드 내용**
- 로봇이 연구실에서 현실 세계로 확장
- 공장·물류·병원·카페·농업·국방
- 안정성·확장성·보안성 요구 급증
- ROS 2 = 현대적 로봇 개발의 필수 기반

**시각적 제안**
- 레이아웃: 좌측 텍스트 + 우측 활용 분야 아이콘 그리드
- 시각 요소: 연구실→현장 전환 일러스트
- 강조: "필수 기반" 키워드 하이라이트

---
### 슬라이드 4: ROS 1의 구조적 한계

**슬라이드 내용**
- 중앙 마스터 의존 → 확장성 제약
- 실시간성 대응 부족
- 보안 기능 미흡
- 멀티 로봇 운영 복잡
- 산업 현장 적용 어려움

**시각적 제안**
- 레이아웃: 5개 카드 세로 리스트
- 시각 요소: 각 한계에 ✕ 아이콘 + 간결 설명
- 강조: 붉은 계열 서브 컬러로 한계 표현

---
### 슬라이드 5: ROS 2가 해결한 핵심 문제

**슬라이드 내용**
- **DDS 분산 통신**: 마스터 없이 노드 직접 연결
- **QoS 지원**: 메시지 전달 정책 세밀 제어
- **보안·안정성**: 서비스 환경 대응 강화
- **멀티 플랫폼**: Linux·Windows·macOS

**시각적 제안**
- 레이아웃: 4분할 카드 (한계→해결)
- 시각 요소: 슬라이드 4의 ✕를 ✓로 전환하는 흐름
- 강조: 각 해결책을 Accent 컬러로 표현

---
### 슬라이드 6: 지금 ROS 2를 선택해야 하는 이유

**슬라이드 내용**
- 미래 지향적 로봇 개발 표준
- 연구에서 산업까지 연결되는 확장성
- 최신 생태계가 ROS 2 중심으로 이동 중
- 멀티 로봇·자율주행에 더 적합

**시각적 제안**
- 레이아웃: 좌측 큰 숫자(1~4) + 우측 설명
- 시각 요소: 타임라인 또는 성장 곡선 그래프
- 강조: "생태계 이동" 키워드 하이라이트

---
### 슬라이드 7: ROS 2가 강한 프로젝트 유형

**슬라이드 내용**
- 자율주행 이동 로봇 — 센서 융합·경로 계획
- 물류 로봇 — 다중 로봇 통신 안정성
- 산업용 로봇 — 정밀 제어·장기 운영
- 서비스 로봇 — 카메라·음성·클라우드 통합

**시각적 제안**
- 레이아웃: 2×2 그리드 카드
- 시각 요소: 프로젝트별 아이콘 + 핵심 강점 라벨
- 강조: 각 카드를 서로 다른 Sub Accent 톤으로 구분

---
### 슬라이드 8: 학습자 관점의 장점

**슬라이드 내용**
- 산업에 가까운 구조를 직접 익힘
- 분산 설계·모듈화 습관 형성
- 센서·제어·시뮬레이션 통합 경험
- 연구·취업·창업으로 확장 가능

**시각적 제안**
- 레이아웃: 좌측 학습자 아이콘 + 우측 장점 리스트
- 시각 요소: 성장 경로 타임라인
- 강조: "장기적 역량" 키워드 하이라이트

---
### 슬라이드 9: 기업·팀 관점의 장점

**슬라이드 내용**
- 기능별 모듈 분리로 협업 구조 개선
- 시뮬레이션 기반 검증 → 리스크 감소
- 시스템 확장성 + 장기 기술 자산화
- 빠른 프로토타이핑과 확장의 균형

**시각적 제안**
- 레이아웃: 좌측 기업 아이콘 + 우측 효과 리스트
- 시각 요소: 팀 협업 다이어그램
- 강조: "기술 자산화" 키워드 하이라이트

---
### 슬라이드 10: 핵심 정리

**슬라이드 내용**
- ROS 2 = 최신 버전이 아닌 구조적 재설계
- 복잡한 시스템·안정적 통신·서비스 환경 대응
- 학습자에겐 역량, 기업에겐 기반 기술
- **미래 로봇 개발 방식에 맞는 기반을 선택하는 일**

**시각적 제안**
- 레이아웃: 중앙 정렬, 4개 핵심 카드
- 시각 요소: 체크 아이콘 + 핵심 문장
- 강조: 마지막 문장을 대형 인용 스타일로 표현
