### 슬라이드 3: ROS 2란 무엇인가

**슬라이드 내용**
- 로봇 소프트웨어 개발용 오픈소스 프레임워크
- OS가 아닌 **미들웨어 기반 플랫폼**
- 센서·모터·통신 기능을 조합해 빠르게 개발
- 반복 구현 대신 구조 위에서 조립

**시각적 제안**
- 레이아웃: 좌측 텍스트 + 우측 다이어그램
- 시각 요소: 로봇 개발 레이어 구조도
- 강조: "미들웨어 플랫폼" 키워드 하이라이트

---
### 슬라이드 4: ROS의 등장 배경

**슬라이드 내용**
- 하드웨어마다 다른 제어 방식
- 소프트웨어 재사용이 어려운 환경
- ROS로 공통 구조·도구 제공 → 재사용성 향상
- ROS 1의 한계 → ROS 2 탄생

**시각적 제안**
- 레이아웃: 전후 비교 (Before / After)
- 시각 요소: 파편화된 개발 → 통합 구조 전환 흐름도
- 강조: "재사용성"과 "확장성" 키워드

---
### 슬라이드 5: ROS 1의 한계

**슬라이드 내용**
- 멀티 로봇 대응 제한
- 실시간성 보장 어려움
- 보안 기능 부족
- 중앙 마스터 구조의 확장성 제약

**시각적 제안**
- 레이아웃: 4분할 카드 레이아웃
- 시각 요소: 각 한계에 ✕ 아이콘 + 간결 설명
- 강조: 붉은 계열 서브 컬러로 한계 표현

### 슬라이드 6: DDS 기반 통신

**슬라이드 내용**
- DDS: 산업용 분산 시스템 통신 표준
- 중앙 마스터 없이 노드 간 직접 통신
- QoS 설정으로 통신 품질 제어
- 멀티 로봇·산업 시스템 연계 용이

**시각적 제안**
- 레이아웃: 중앙 다이어그램 + 하단 키포인트
- 시각 요소: 분산 노드 네트워크 다이어그램
- 강조: "DDS"를 대형 라벨로 표현

---
### 슬라이드 7: 노드 기반 구조와 메시지 통신

**슬라이드 내용**
- 기능 단위 = **노드**(카메라, 라이다, 모터)
- 독립 실행 → 유지보수·확장 용이
- Topic / Service / Action 통신 방식
- 느슨한 결합으로 유연한 설계

**시각적 제안**
- 레이아웃: 상단 노드 구조도 + 하단 통신 비교 표
- 시각 요소: 노드 연결 다이어그램, 3열 비교 카드
- 강조: Topic·Service·Action을 색상으로 구분

### 슬라이드 8: ROS 2 활용 분야

**슬라이드 내용**
- 자율주행·물류·서비스 로봇
- 산업용 로봇 팔·스마트 팩토리
- 드론 및 무인 이동체
- 교육·연구용 플랫폼

**시각적 제안**
- 레이아웃: 2×4 그리드 아이콘 카드
- 시각 요소: 분야별 아이콘 + 한 줄 설명
- 강조: 산업 적용 영역을 Accent 컬러로 표현

---
### 슬라이드 9: ROS 1 vs ROS 2

**슬라이드 내용**
- 통신: Master 기반 → DDS 분산 통신
- 실시간: 제한적 → 개선
- 보안: 부족 → 확장 가능
- OS: Linux 중심 → 멀티 OS 지원

**시각적 제안**
- 레이아웃: 2열 비교 테이블
- 시각 요소: ROS 1(회색) vs ROS 2(Accent) 대비
- 강조: ROS 2 열을 Accent 배경으로 강조

### 슬라이드 10: 핵심 정리

**슬라이드 내용**
- ROS 2 = 현대적 로봇 개발 오픈소스 프레임워크
- DDS 분산 통신 + 노드 기반 모듈 구조
- 시뮬레이션과 실제 로봇 연결 용이
- **로봇 시스템을 구조적으로 설계하는 방법을 배우는 것**

**시각적 제안**
- 레이아웃: 중앙 정렬, 4개 핵심 카드
- 시각 요소: 체크 아이콘 + 핵심 문장
- 강조: 마지막 문장을 대형 인용 스타일로 표현
