### 슬라이드 3: 운영체제 선택과 사전 준비

**슬라이드 내용**
- Ubuntu Linux가 가장 안정적 환경
- 공식 문서·커뮤니티·사례 모두 풍부
- HW 권장: 64비트 CPU, RAM 8GB+
- SW 준비: Python, Git, 터미널 사용

**시각적 제안**
- 레이아웃: 좌측 Ubuntu 로고 + 우측 체크리스트
- 시각 요소: HW/SW 요구사항 카드
- 강조: "Ubuntu 우선 추천" 배지

---
### 슬라이드 4: ROS 2 설치 방식 비교

**슬라이드 내용**
- **패키지 매니저**: 가장 간편, 초보자 추천
- **소스 빌드**: 커스터마이징, 난이도 높음
- **Docker**: 환경 격리, 배포에 유리
- 학습 시작은 패키지 매니저 설치가 최적

**시각적 제안**
- 레이아웃: 3열 비교 카드
- 시각 요소: 난이도 표시 게이지 (쉬움/중간/높음)
- 강조: 패키지 매니저 카드를 Accent 테두리로 강조

### 슬라이드 5: 필수 개발 도구

**슬라이드 내용**
- Git / Python3 / pip — 기본 개발
- colcon / rosdep / vcstool — ROS 2 빌드
- VS Code — 코드 편집 + 확장 프로그램
- tmux — 멀티 터미널 세션 관리

**시각적 제안**
- 레이아웃: 2열 그리드 (필수 vs 부가)
- 시각 요소: 도구별 아이콘 + 한 줄 역할 설명
- 강조: 필수 도구를 Accent 배경으로 표현

---
### 슬라이드 6: 환경 변수 설정

**슬라이드 내용**
- `source` 명령으로 ROS 2 설정 로드
- 워크스페이스 install 경로도 별도 로드
- 쉘 설정 파일에 등록하면 자동 반영
- 미설정 시 명령어·패키지 인식 불가

**시각적 제안**
- 레이아웃: 터미널 화면 모의 + 하단 주의사항
- 시각 요소: 코드 블록 스타일 `source` 명령 예시
- 강조: ⚠️ "미설정 시 오류" 경고 배지

---
### 슬라이드 7: 워크스페이스 구조

**슬라이드 내용**
- `src/` — 소스코드·패키지 저장
- `build/` — 빌드 중간 파일
- `install/` — 실행 가능 결과물
- `log/` — 빌드·실행 로그

**시각적 제안**
- 레이아웃: 중앙 디렉터리 트리 다이어그램
- 시각 요소: 폴더 아이콘 + 역할 라벨
- 강조: `src` 폴더를 Accent 컬러로 강조

---
### 슬라이드 8: 첫 패키지 생성과 개발 사이클

**슬라이드 내용**
- 워크스페이스 생성 → 패키지 생성 → 빌드
- source 적용 → 노드 실행 → 로그 확인
- 코드 수정 → 재빌드의 반복 구조
- 이 사이클이 ROS 2 개발의 핵심 습관

**시각적 제안**
- 레이아웃: 순환형 플로우차트
- 시각 요소: 6단계 원형 화살표 다이어그램
- 강조: "반복 사이클" 키워드 하이라이트

---
### 슬라이드 9: 시뮬레이션 환경과 초기 문제

**슬라이드 내용**
- Gazebo: 물리 기반 시뮬레이션
- RViz2: 센서·좌표계 시각화
- 흔한 오류: 환경 변수 누락, source 미적용
- 대부분 환경 설정 이해로 해결 가능

**시각적 제안**
- 레이아웃: 좌측 시뮬레이션 도구 + 우측 트러블슈팅 카드
- 시각 요소: Gazebo/RViz 스크린샷 모의 + ❌→✅ 해결 흐름
- 강조: "환경 설정 이해" 키워드

---
### 슬라이드 10: 핵심 정리

**슬라이드 내용**
- OS 선택 + 도구 설치 + 환경 변수 = 기반 완성
- 워크스페이스 구조와 빌드 사이클 숙지
- 시뮬레이터로 실습 → 실제 로봇 연결
- **환경이 잘 갖춰지면 ROS 2는 친숙한 도구가 된다**

**시각적 제안**
- 레이아웃: 중앙 정렬, 4개 핵심 카드
- 시각 요소: 체크 아이콘 + 핵심 문장
- 강조: 마지막 문장을 대형 인용 스타일로 표현
