**시각적 제안**

### 슬라이드 7: 메시지 타입과 패키지

**슬라이드 내용**
- 메시지 = 노드 간 데이터 전달 형식
- 대표: `Twist`, `Image`, `LaserScan`
- 패키지 = 프로젝트의 기본 단위
- 의존성 관리로 안정성 확보

**시각적 제안**
- 레이아웃: 상단 메시지 예시 + 하단 패키지 구조도
- 시각 요소: 코드 블록 스타일 메시지 예시
- 강조: `Twist` 메시지 구조를 인포그래픽으로 표현

---
### 슬라이드 8: 좌표계와 TF / 센서·액추에이터

**슬라이드 내용**
- 좌표계: `map`, `odom`, `base_link`
- TF로 좌표계 간 관계 계산·관리
- 센서(카메라·라이다·IMU) = 입력
- 액추에이터(모터·서보) = 출력

**시각적 제안**
- 레이아웃: 좌측 TF 트리 다이어그램 + 우측 센서/액추에이터 리스트
- 시각 요소: 좌표 프레임 연결 다이어그램
- 강조: 입력(파란색)과 출력(주황색) 색상 구분

---
### 슬라이드 9: 빌드 시스템과 디버깅

**슬라이드 내용**
- `colcon` 빌드 도구로 워크스페이스 관리
- 구조: `src` → `build` → `install`
- 디버깅 CLI: `ros2 node/topic/service list`
- 로그 읽기 + 에러 구분 습관 중요

**시각적 제안**
- 레이아웃: 상단 빌드 프로세스 흐름도 + 하단 CLI 명령어 카드
- 시각 요소: 워크스페이스 디렉터리 트리 다이어그램
- 강조: `colcon build` 명령어를 코드 스타일로 표현
